# guide_integration.md — 통합 참고서

## 데이터 흐름

### 1. 양식 선택
- 프론트 → 백엔드: 선택된 양식 ID (체크박스 기반)
- 백엔드 → 프론트: 양식 메타데이터

### 2. 업로드
- 프론트 → 백엔드: 다중 이미지 파일 (드래그앤드랍 또는 선택)
- 백엔드: 임시 저장

### 3. 인식
- 프론트 → 백엔드: 인식 요청 (업로드 세션 ID)
- 백엔드 → 모델: 이미지 전달 (필요 시 detection 후 인식)
- 모델 → 백엔드: 텍스트, bbox, confidence
- 백엔드 → 프론트:
  - 이미지별 인식 결과
  - bbox 좌표 (라운드박스 렌더링용)
  - 저신뢰도 플래그 + **후보 텍스트 리스트**
  - 양식에 채워진 필드 값

### 4. 저장
- 프론트 → 백엔드: 최종 수정된 양식 내용 + 민원인 생년월일
- 백엔드: 암호화 후 저장소(임시 DB / Google Drive)에 기록

---

## 점검 사항
- [ ] bbox 좌표계 통일 (원본 이미지 기준 / 리사이즈 이미지 기준)
- [ ] Confidence threshold 일관성
  - 백엔드가 판정 후 플래그만 전달할지
  - 프론트가 raw 점수를 받아 판정할지
- [ ] 인식 실패 / 부분 실패 처리 정책
- [ ] Detection 단계 위치 — `guide_model.md` 참조
- [ ] 양식의 어느 입력 필드에 어떤 인식 결과를 매핑할지의 규칙 — **현재 미명시, 결정 필요**
- [ ] 프론트에서 사용자가 수정한 텍스트를 저장 시 어떻게 백엔드와 양식에 반영할지
