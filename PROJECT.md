# 개발 가이드 라인
## 사용 과정
1. 민원인이 정해진 서류를 직접 손으로 작성
2. 공무원은 서류들을 스캔해서 pc로 옮김
3. (공무원 내부망에 publish된 웹서버라고 가정) pc에 있는 민원인 작성 서류 스캔본을 웹사이트에 파일 양식 선택(클릭) 후 업로드
4. 업로드된 파일들의 손글씨들을 인식해서 글자로 변환해서 선택한 파일 양식으로 저장
4-1. 업로드 된 이미지에서 손글씨 인식된 부분은 파란색으로 라운드박스 or 밑줄 표시해서 화면 좌측 표시
4-2. 인식 score 낮은(확신이 잘 안 드는) 손글씨는 빨간색으로 라운드박스 or 밑줄 표시하고 바로 아래에 아이폰 텍스트 제안처럼 여러 후보 제시
5. 전산업무가 끝나면 민원인 정보 무단열람을 막기 위해 파일 개인정보 마스킹 처리 or 잠금 설정(생년월일로만 해놔도 실수로 열었다는 말 못함)


# 풀스택 개발 계획 draft
## 프론트엔드
- 초기 화면은 좌측에는 파일 업로드하는 공간, 우측은 양식 선택 및 검색하는 공간(체크박스 있어야됨)
- 좌측에 드래그앤드랍 or 파일선택(다중 업로드 가능해야함)해서 올리면 중앙 하단에 인식 버튼이 회색 -> 파란색으로 변환
- 인식버튼 누르면 진행 progress bar랑 밑에 진행 % 표시
- 인식 다 되면 좌측에는 업로드한 이미지들과 손글씨 인식부분 라운드 박스 등이 보이고, 우측에는 선택한 양식에 입력된 파일의 미리보기 생성 및 직접 타이핑으로 수정 가능한 상태로 보여줌. 여러 이미지인 경우 이미지 하단에 다른 이미지 볼 수 있도록 사진 넘기기 기능(좌우 화살표 및 번호) 있음
- 작업(정정사항 수정 및 본인확인 등)이 끝나면 우측 양식대로 전산에 저장(우리는 임시 db나 구글드라이브 활용)

## 백엔드
- 양식 목록은 DB에 저장된 양식 파일들을 동적으로 받아와서 목록 보여줌. 즉, DB에 양식 파일 추가만 하면 다음에 해당 사이트 접속 시 바로 보여주고 선택도 가능
- 초기에 양식 선택하면 해당 양식 로드 및 인식된 텍스트 받을 준비
- 이미지 업로드된 후 인식 버튼 누르면 학습된 모델로 손글씨를 인식해서 텍스트로 변환 후 지정된 양식에 입력
- 손글씨 부분으로 인식된 부분은 라운드박스 처리 및 inference loss 값 높은 부분은 밑에 텍스트 제안처럼 표기
- 작업 완료 후 화면 하단의 저장 버튼 클릭 시 전산(우리 프로젝트에서는 구글드라이브나 임시 DB활용)에 파일 암호화(초기 암호는 민원인 생년월일) 저장


# 모델링 계획 draft
## 모델링 방향
### mvp
1. ddobokki/ko-trocr(pre-trained model)을 사용해서 zero-shot 테스트
2. ddobokki/ko-trocr에 일반 손글씨 데이터로 fine-tuning
3. 이후 노인 손글씨 데이터로 약간(1번에서 fine-tuning에 사용한 데이터 양보다 적게) 부가 fine-tuning (아직 진행x)

### development
1. 일반 손글씨, 노인 손글씨 데이터 data augmentation(geometric transformation, elastic distortion, noise injection 등) 진행 후 fine-tuning 처음부터 재진행해서 robust한 모델 확립
2. 손글씨 인식 DL/ML 관련 논문 찾아보고 성능 향상
